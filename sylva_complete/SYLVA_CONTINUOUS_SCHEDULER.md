## 🎯 系统架构

### 核心原则
1. **永不停止**: 一旦启动，持续运转直到用户明确喊停
2. **边答边运**: 回答用户的同时，后台Agent集群持续工作
3. **自我诊断**: 遇到卡顿自动触发skill诊断修复
4. **任务连续**: 完成一项自动启动下一项，无缝衔接

### 运行模式
```
┌─────────────────────────────────────────────────────────────┐
│                     主会话 (Main Session)                     │
│  - 响应用户查询                                               │
│  - 调度Agent集群                                              │
│  - 监控系统健康                                               │
└──────────────┬──────────────────────────────────────────────┘
               │ 任务分派
               ▼
┌─────────────────────────────────────────────────────────────┐
│                   Agent集群 (Subagent Pool)                   │
│  - 并行处理多项任务                                           │
│  - 每任务独立会话                                             │
│  - 结果自动回传                                               │
└──────────────┬──────────────────────────────────────────────┘
               │ 状态反馈
               ▼
┌─────────────────────────────────────────────────────────────┐
│                   自我诊断层 (Self-Diagnosis)                 │
│  - 检测卡顿/错误                                              │
│  - 自动调用skill修复                                          │
│  - 记录故障日志                                               │
└─────────────────────────────────────────────────────────────┘
```

## 🔄 连续运作协议

### 1. 任务完成触发
当Agent完成任务时:
```
任务完成信号
    ↓
自动检查等待队列
    ↓
启动下一优先级任务
    ↓
更新状态看板
    ↓
向用户简要汇报
```

### 2. 卡顿检测机制
```yaml
卡顿指标:
  - 响应延迟 > 30秒
  - 工具调用失败 > 3次
  - 编译错误持续 > 5分钟
  - 用户指令未响应

自动修复流程:
  1. 触发诊断skill检查
  2. 尝试自我修复 (edit/patch)
  3. 若失败 → 启动修复Agent
  4. 记录到故障日志
```

### 3. 资源监控
```yaml
监控项:
  - Token消耗率 (每分钟)
  - Agent并发数
  - 内存使用率
  - 编译状态

阈值告警:
  - Token > 1000/min → 节流
  - Agent > 10并发 → 排队
  - 内存 > 80% → 清理
```

## 🛠️ 自我修复技能清单

### 编译错误修复
| 错误类型 | 诊断Skill | 修复策略 |
|---------|----------|---------|
| Lean编译失败 | - | 截肢降级 → 证明填充 |
| 类型不匹配 | - | 类型推导修复 |
| 缺失导入 | - | 自动添加依赖 |
| sorry填充 | - | 启动证明Agent |

### 系统卡顿修复
| 卡顿类型 | 诊断方法 | 修复策略 |
|---------|---------|---------|
| 工具超时 | 检查exec状态 | 重试/调整超时 |
| 内存不足 | 检查资源使用 | 压缩/清理历史 |
| 网络中断 | 检查连接状态 | 自动重连 |
| 循环依赖 | 检查调用栈 | 打破循环 |
