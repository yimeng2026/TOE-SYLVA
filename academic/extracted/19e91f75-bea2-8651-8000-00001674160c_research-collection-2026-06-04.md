## 方向3：千界花园 Electron 打包与构建相关

### 核心工具
1. **electron-builder** (官方)
   - 一键打包 macOS/Windows/Linux
   - 内置 auto-updater（electron-updater）
   - 代码签名与公证（macOS notarization + Windows Authenticode）
   - URL: https://www.electron.build/

2. **Electron Forge** (官方推荐)
   - 电池包含工具包
   - `npm init electron-app@latest`
   - 支持 React/Vue/Next.js/Tailwind/Three.js 等
   - URL: https://electronjs.org/

3. **CI/CD 代码签名** (Katson, 2024-02-27)
   - Azure Key Vault + GlobalSign 方案
   - 2024 年规则变化：不再允许导出 PFX，需 HSM 或云签名
   - URL: https://katson.com/code-signing-in-2024/

4. **Sectigo SafeNet Token 自动化** (Xojo Forum, 2024-11)
   - USB 硬件令牌 + SignTool 自动化
   - 单点登录模式 vs 完全自动化（密码嵌入批处理）
   - URL: https://forum.xojo.com/t/sectigo-code-signing-certificate-2024-edition/83332

5. **安全警告：CVE-2024-39698**
   - electron-updater < 6.3.0-alpha.6 存在 Windows 签名验证绕过
   - cmd.exe 环境变量解析导致验证错误文件
   - URL: https://cve.imfht.com/detail/CVE-2024-39698

### 构建建议
- 使用 electron-builder 或 Electron Forge
- 代码签名需准备 HSM 或云方案（2024 年后 PFX 导出已受限）
- auto-updater 必须升级到 >= 6.3.0-alpha.6
- GitHub Actions 或 GitLab CI 可自动化构建流水线

## 方向4：论文撰写系统 / 幻觉检验系统 的七阶段流水线文献

### 核心文献
1. **PaperOrchestrator** (Springer, 2026-04-06)
   - 七阶段：outline → section-by-section generation → length control → consistency polishing → BibTeX → LaTeX conversion → journal template rendering
   - 共享上下文数据结构管理阶段间信息传递
   - BERTScore F1: 0.674, LaTeX 编译率: 90%
   - URL: https://link.springer.com/article/10.1007/s44443-026-00708-4

2. **Suprmind Multi-LLM Workflow** (2026-04-20)
   - 七步：Define question → Gather sources → Parallel reading → Debate synthesis → Adjudicate facts → Draft sections → Final checks
   - 多模型共识防止幻觉
   - URL: https://suprmind.ai/hub/insights/best-ai-for-writing-research-papers-a-multi-llm-workflow-that-holds/

3. **VaaS (Validation as a System)** (medRxiv, 2026-03-30)
   - 多层幻觉削减流水线
   - 认识论完整性约束（"The First Law"）：禁止生成未验证引用
   - 实时 PMID 获取 → 主题验证 → 修正列表注入
   - 640 次对照运行验证
   - URL: https://www.medrxiv.org/content/10.64898/2026.03.24.26348935v1.full

4. **EH-Benchmark** (2025-05-20)
   - 三阶段多 Agent 框架：Knowledge-Level Retrieval → Task-Level Case Studies → Result-Level Validation
   - 27K 眼科问题，评估 Visual Understanding + Logical Composition 幻觉
   - 正确性、完整性、工作流遵循度三维评估
   - URL: https://arxiv.org/html/2507.22929v1

5. **Wildflare GuardRail** (arXiv, 2025-02)
   - 四阶段护栏：检测 → 上下文锚定 → 输出修正 → 用户自定义
   - 可解释幻觉检测模型（Fox-1 微调）
   - 全知识索引 + 关键信息索引
   - URL: https://arxiv.org/html/2502.08142v1

6. **WisPaper** (arXiv, 2026-04-13)
   - 端到端学术 Agent：Scholar Search → Library → AI Feeds
   - WisModel：RL 训练，分解查询为可验证标准，要求原文直接引用作为证据
   - 消除幻觉
   - URL: https://arxiv.org/html/2512.06879v3

## 方向5：Agent协作架构（六层嵌套、可打断/开会/接力机制）开源参考

### 架构模式分类
1. **六种核心拓扑** (System Design Newsletter, 2026-04-30)
   - Orchestrator-worker、Pipeline、Hierarchical、Swarm、Mesh、Handoffs
   - Anthropic Claude Research：Orchestrator-worker，Opus 4 调度 2-10 Sonnet 工人，并行搜索
   - 比单 Agent 提升 90.2%
   - URL: https://newsletter.systemdesign.one/p/multi-agent-system

2. **四种基础模式** (LangChain, 2026-01-14)
   - Subagents（中心协调）、Skills（按需加载）、Handoffs（状态驱动转移）、Routers（路由）
   - Handoffs 最适合作业流程：状态存活、顺序约束、能力解锁
   - URL: https://www.langchain.com/blog/choosing-the-right-multi-agent-architecture

3. **AgentOrchestra** (arXiv, 2025-05-13)
   - 分层规划 + 模块化子 Agent
   - 统一工具接口支持文本/图像/音频/视频/结构化数据
   - 动态角色分配
   - URL: https://arxiv.org/html/2506.12508v1

4. **Hermes Agent 多 Agent 路线图** (GitHub, 2026-03)
   - 三阶段：Phase 1 Workflow DAG + Synthesis → Phase 2 Resilient Execution → Phase 3 Agent Roles & Cooperation
   - 失败恢复：Retry → Replan → Decompose
   - 四级上下文共享：L0 隔离 → L1 结果传递 → L2 共享暂存 → L3 实时对话
   - 借鉴：Gas Town (Steve Yegge)、CAMEL-AI (NeurIPS 2023)
   - URL: https://github.com/NousResearch/hermes-agent/issues/344

5. **Gas Town** (Steve Yegge)
   - 20-50+ 并发编码 Agent 的 Go 系统
   - 工作流公式：Convoy（并行+合成）、Workflow（顺序+依赖）、Expansion（模板）、Aspect（AOP）
   - GUPP："If you find work on your hook, YOU RUN IT"
   - 三级健康监控：Daemon → Boot → Deacon
   - URL: https://github.com/steveyegge/gastown

6. **CAMEL-AI / Eigent** (NeurIPS 2023)
   - Workforce 5 步生命周期：Decompose → Assign → Execute → Complete → Handle Failures
   - LLM-based 协调器路由（非规则）
   - Agent 池：每种 Worker 10 个预创建克隆
   - Inception Prompting：防止多 Agent 通信失败
   - URL: https://github.com/camel-ai/camel

7. **协作结构形式化** (Agentic AI in Financial Services, 2025)
   - Horizontal Collaboration（水平）、Hierarchical（层级）、Nested/Hybrid（嵌套/混合）
   - 动态结构：Agent 角色、数量、关系可演化
   - URL: https://arxiv.org/pdf/2502.05439v1
