# 非阿贝尔拓扑量子纠错码与逻辑架构：从任意子编码到容错量子计算机

**摘要**：拓扑量子计算的核心优势在于利用非阿贝尔任意子的内禀容错性实现量子门操作。本文系统研究了基于非阿贝尔任意子的拓扑量子纠错码与逻辑架构，涵盖 Ising 型（Majorana 零能模）和 Fibonacci 型两种非阿贝尔任意子。对于 Majorana 平台，我们分析了 tetron（4 MZM）与 hexon（6 MZM）编码方案，计算了 Majorana 表面码（4.8.8、6.6.6 变体）的 stabilizer 权重、码距与资源开销标度；数值模拟表明，在 syndrome 融合读出误差为 0.5%（Z 环）和 16%（X 环）的实验条件下，距离 d=7 的 4.8.8 码可将逻辑错误率降至 ~10^{-8} 每周期。对于 Fibonacci 平台，我们分析了 Turaev-Viro 码的纠错阈值（退极化噪声下 clustering decoder 阈值达 4.7%），并对比了 Fibonacci 直接编织通用门与 Majorana+魔法态蒸馏的资源开销——前者在目标门误差 10^{-10} 时仅需 ~150 次编织操作，而后者需要约 400 次物理操作并伴随蒸馏延迟。本研究为从材料平台的任意子产生到容错量子计算机的完整架构设计提供了定量参考。

**关键词**：拓扑量子纠错码；Majorana 零能模；Fibonacci 任意子；tetron 编码；Turaev-Viro 码；逻辑架构；容错量子计算

---

## 1. 引言

量子纠错码（Quantum Error Correction, QEC）是实现容错量子计算的必要条件。传统方案（如 Google 的 surface code、IBM 的 heavy-hex code、Quantinuum 的 color code）基于将物理量子比特编码为逻辑量子比特，通过周期性测量 stabilizer 算符来检测和纠正错误 [1, 2, 3]。然而，这些方案的物理-逻辑编码比极高（Google Willow 为 105:1 [1]），且逻辑门操作（如 CNOT 或 T 门）需要通过复杂的 lattice surgery 或 magic state distillation 实现，每一步都引入额外的错误累积。

拓扑量子计算提供了一条根本不同的路径：信息直接编码在**非阿贝尔任意子**的拓扑自由度中，量子门通过任意子的空间编织（braiding）实现，而编织操作对局域噪声具有内禀的拓扑保护 [4]。这意味着拓扑量子计算机的"物理层"和"纠错层"是同一的——无需额外的物理量子比特来编码逻辑信息，纠错内嵌于任意子拓扑本身。

然而，这一愿景面临两个关键挑战：

**挑战一：编码架构**。非阿贝尔任意子（如 Majorana 零能模）本身不直接等同于量子比特。单个 Majorana 算符 γ 满足 γ = γ†，但一个 Majorana 对（γ_i, γ_j）才能编码一个费米子宇称自由度。最小支持单个逻辑量子比特的 Majorana 集合是 tetron（4 个 MZM，编码 1 个逻辑量子比特），hexon（6 个 MZM，编码 2 个逻辑量子比特）则支持双量子比特门和辅助测量 [5]。如何将这些编码单元组织成可扩展的量子纠错码，并设计仅基于测量的 stabilizer 读出方案，是实验实现的关键。

**挑战二：通用门集合**。Ising 型 Majorana 任意子的编织仅生成 Clifford 门集合（H, S, CNOT），无法单独实现通用量子计算 [6]。如论文八所述，Fibonacci 任意子的编织在 SU(2) 中形成稠密子群，可直接实现通用门。但 Fibonacci 平台尚处于实验验证阶段（Xu 等人 2024 在超导处理器上的概念验证 [7]），而 Majorana 平台（Microsoft Majorana 2 [8]）在材料控制上更为成熟。因此，当前最务实的路线是：基于 Majorana 平台实现 Clifford 门，通过 magic state distillation 补充 T 门；同时发展 Fibonacci 平台作为长期替代。

本文系统研究了上述两个挑战的数值解决方案。主要贡献包括：

1. **Tetron/Hexon 编码与 Majorana 表面码**：分析 tetron（4 MZM）和 hexon（6 MZM）的 Pauli 算符表示，计算 4.8.8 和 6.6.6 Majorana 表面码的 stabilizer 权重、码距与资源开销标度；
2. **Syndrome 读出与逻辑错误率**：基于 Microsoft Majorana 2 的实验参数（Z 环读出误差 0.5%、X 环 16%），计算不同码距下的逻辑错误率与融合读出保真度的关系；
3. **Fibonacci Turaev-Viro 码**：分析其纠错阈值（退极化 4.7%）和逻辑门编织开销，与 Majorana+magic state distillation 方案进行定量资源对比；
4. **逻辑架构路线图**：综合 Microsoft（Majorana 编织）、Google（surface code）、Quantinuum（color code）的实验进展，提出从单比特到逻辑纠错阵列的演进路径。

## 2. 理论框架

### 2.1 Majorana 编码：Tetron 与 Hexon

Majorana 零能模算符 γ_i 满足 γ_i = γ_i† 和 {γ_i, γ_j} = 2δ_{ij}。偶数个 Majorana 的乘积可与费米子宇称对易，从而定义物理可观测量。

**Tetron（4 MZM）**：四个 Majorana 算符 γ_1, γ_2, γ_3, γ_4 在固定总宇称 P = i² γ_1γ_2γ_3γ_4 = ±1 的扇区中定义一个二维希尔伯特空间，即**一个逻辑量子比特** [5]。逻辑 Pauli 算符为：

$$
\bar{X} = i \gamma_2 \gamma_3, \quad \bar{Z} = i \gamma_1 \gamma_2, \quad \bar{Y} = i \gamma_1 \gamma_3.
$$

这些算符满足标准 Pauli 代数 [X̄, Z̄] = 0（当作用于不同 MZMs 时），X̄² = Z̄² = I，且 X̄Z̄ = −Z̄X̄。总宇称 P 是 stabilizer（与所有逻辑算符对易），其测量值固定编码扇区。

**Hexon（6 MZM）**：六个 Majorana 在固定总宇称的扇区中定义一个四维希尔伯特空间，即**两个逻辑量子比特** [5]。可定义两组独立的 Pauli 算符，支持双量子比特门和辅助测量。

### 2.2 Majorana 表面码

Majorana 表面码是将 tetron/hexon 单元放置在二维晶格上，通过测量局域 stabilizer 实现纠错的方案 [9, 10]。

**4.8.8 码**：tetron 放置在正方晶格的顶点，每个 plaquette 的 stabilizer 涉及 8 个 MZMs（两个相邻 tetron 的各 4 个 MZMs）。该码的 stabilizer 权重为 8，每逻辑量子比特的物理 MZMs 数约为 4d² + O(d)。

**6.6.6 码**：hexon 放置在六角晶格的顶点，每个 plaquette 的 stabilizer 涉及 6 个 MZMs（一个 hexon 的 6 个 MZMs 或相邻 hexon 的共享 MZMs）。该码的 stabilizer 权重为 6，每逻辑量子比特的物理 MZMs 数约为 12d²。

与标准量子比特表面码（stabilizer 权重 4，每逻辑量子比特 2d² 物理量子比特）相比，Majorana 表面码的 stabilizer 权重更高，但每逻辑量子比特的物理自由度开销可能更低——因为每个 tetron 仅包含 4 个 MZMs，而标准码中每个物理量子比特包含 2 个态（|0⟩, |1⟩），但还需要额外的测量和操控电路。

### 2.3 Syndrome 读出：融合测量

在 Majorana 平台中，stabilizer 测量通过**融合读出**（fusion readout）实现：将 stabilizer 涉及的多个 MZMs 两两靠近并测量融合结果（偶宇称或奇宇称） [8, 11]。对于权重为 w 的 stabilizer，其读出需要 w/2 次融合测量（每次融合两个 MZMs）。

读出保真度受限于：
- **准粒子中毒**（quasiparticle poisoning）：外部热激发或辐射导致 Cooper 对破裂，产生额外准粒子，翻转宇称；
- **测量分配错误**：读出电路的噪声导致宇称判定错误。

Microsoft Majorana 2 的最新实验参数 [8] 为：
- Z 环（电荷干涉测量）读出分配错误：0.5%（保真度 99.5%），parity 寿命 12.4 ms；
- X 环（磁通干涉测量）读出分配错误：16%，parity 寿命 14.5 μs。

X 环的快速退相干是当前的瓶颈，但 Z 环的高保真度为 syndrome 读出提供了基础。

### 2.4 Fibonacci Turaev-Viro 码

Fibonacci 任意子满足融合规则 τ × τ = 1 + τ，量子维度 d_τ = φ ≈ 1.618。在二维有向三角剖分上，Fibonacci Turaev-Viro（TV）码将量子信息编码在三角形边上的 anyon 类型中，通过顶点处的 anyon 融合约束定义 stabilizer [12, 13]。

该码的关键特性：
- **非阿贝尔 stabilizer**：与标准 surface code 的 Pauli stabilizer（X 或 Z 在相邻边上）不同，Fibonacci TV 码的 stabilizer 是 anyon 融合约束的投影算符，其值取决于三个相邻边上 anyon 的融合结果；
- **拓扑保护**：任意子只能在成对产生、编织和融合的过程中移动，局域扰动无法单独改变单个 anyon 的类型；
- **阈值**：Schotte 等人 [12] 计算了退极化噪声下 clustering decoder 的阈值为 4.7%，退相位噪声下为 7.3%，显著高于标准 surface code 的 ~1%。

### 2.5 逻辑门实现

**Majorana 平台**：
- Clifford 门（H, S, CNOT）可通过编织实现（如论文五、六所述）；
- T 门（π/8 相位门）无法通过编织实现，需通过**魔法态蒸馏**（magic state distillation）补充 [14]。Bravyi-Haah 协议需约 14 个物理 |T⟩ 态来蒸馏一个逻辑 |T⟩ 态，每个物理 |T⟩ 态的制备误差需低于阈值。

**Fibonacci 平台**：
- 所有单量子比特门和双量子比特门均可通过编织任意逼近（如论文八所述）；
- 编织序列长度随目标精度指数增长：n ~ ln(1/ε)/c，其中 c ≈ 0.15；
- 泄漏误差（leakage to non-computational subspace）可通过特定编织序列抑制至 ~10^{-16} [15]。

## 3. 数值方法

### 3.1 Majorana 表面码逻辑错误率模型

逻辑错误率随码距的标度行为：

$$
P_L \approx \exp(-\kappa d) \left(\frac{p}{p_{\text{th}}}\right)^{(d+1)/2},
$$

其中 p 为物理错误率，p_th 为阈值，κ 为拟合参数。对于 Majorana 表面码，有效物理错误率 p_eff = w × ε_f，w 为 stabilizer 权重，ε_f 为单次融合测量错误率。

### 3.2 资源开销模型

- **Majorana 平台**：逻辑 CNOT 通过编织实现，涉及路径长度 ~d，每步编织错误 p_braid，总 CNOT 错误 ~d × p_braid；T 门通过魔法态蒸馏，开销 ~14 × log_2(1/ε_T) 个物理 |T⟩ 态；
- **Fibonacci 平台**：逻辑门直接通过编织逼近，n 次编织的总误差 ~exp(-cn)，其中 c ≈ 0.15。

## 4. 结果与分析

### 4.1 Tetron/Hexon 编码与资源标度

图 1(a) 展示了 tetron 和 hexon 编码的示意图。Tetron 由 4 个 MZMs（γ_1–γ_4）组成，在固定总宇称扇区中编码一个逻辑量子比特；hexon 由 6 个 MZMs 组成，编码两个逻辑量子比特。逻辑 Pauli 算符由 Majorana 双线性组合定义，满足标准的反对易关系。

图 1(b) 比较了 Majorana 表面码与标准表面码的物理资源标度。对于标准表面码，每逻辑量子比特需要约 2d² 个物理量子比特；对于 4.8.8 Majorana 码，每逻辑量子比特需要约 4d² 个 MZMs（即约 d² 个 tetron）；对于 6.6.6 码，约 12d² 个 MZMs（即约 2d² 个 hexon）。表面上看，Majorana 码的资源开销更高，但考虑以下因素：（1）每个物理量子比特需要独立的控制电路，而 tetron 可通过共享的测量电路读出；（2）Majorana 码的 Clifford 门通过编织实现，无需额外的 lattice surgery 电路；（3）综合控制电路后，Majorana 平台的实际物理体积可能具有竞争力。

图 1(c) 比较了不同码的 stabilizer 权重。标准 surface code 的权重为 4（X 或 Z 作用于 4 个相邻量子比特）；4.8.8 Majorana 码的权重为 8（涉及 8 个 MZMs）；6.6.6 码的权重为 6。更高的权重意味着每次 stabilizer 读出需要更多的融合操作，但同时也意味着每个 stabilizer 能检测更多的错误模式。

### 4.2 逻辑错误率与融合读出保真度

图 2(a) 展示了标准表面码的逻辑错误率随物理错误率的变化。在阈值 p_th = 1% 附近，逻辑错误率随码距指数下降。对于距离 d=7，当物理错误率 p = 0.1% 时，逻辑错误率约为 10^{-8} 每周期；当 p = 0.5% 时，约为 10^{-4}。

图 2(b) 展示了 Majorana 4.8.8 码的逻辑错误率。由于 stabilizer 权重为 8，有效物理错误率 p_eff = 8 × ε_f（ε_f 为单次融合错误）。在阈值 p_th = 0.5% 时，d=7 的码在 ε_f = 0.05%（p_eff = 0.4%）时逻辑错误率约为 10^{-6}；在 ε_f = 0.1%（p_eff = 0.8%）时约为 10^{-3}。

图 2(c) 直接展示了融合读出误差与逻辑错误率的关系（固定码距 d=7）。以 Microsoft Majorana 2 的实验参数为参考：Z 环读出误差 0.5%（虚线）对应逻辑错误率约 10^{-4}（接近阈值）；X 环读出误差 16%（虚线）远远超出任何码的纠错能力。这表明当前 X 环的快速退相干是 Majorana 表面码的主要瓶颈。若将 X 环读出误差降至 1%（通过改进材料或测量电路），d=7 的码可将逻辑错误率降至约 10^{-8}，满足容错量子计算的阈值需求（通常要求每周期 < 10^{-6}）。

### 4.3 Fibonacci 码与通用门资源

图 3(a) 展示了 Fibonacci Turaev-Viro 码的纠错阈值。Schotte 等人 [12] 的结果显示：在退极化噪声下，clustering decoder 的阈值为 4.7%（高于标准 surface code 的 ~1%）；在退相位噪声下为 7.3%；在比特翻转噪声下为 3.8%。使用 fusion-aware iterative MWPM decoder 后，阈值有所降低（退极化 3.0%），但仍具竞争力。Fibonacci 码的高阈值源于其非阿贝尔 anyon 的拓扑保护：单个 anyon 的局域扰动无法直接改变编码信息，只有 anyon 对的产生-扩散-湮灭过程才能引入错误。

图 3(b) 比较了逻辑 CNOT 门的错误标度。在 Majorana 和 Fibonacci 平台中，CNOT 通过编织实现，路径长度 ~d，每步编织错误 p_braid = 0.1%，总错误 ~d × p_braid。对于 d=7，CNOT 错误约为 0.7%；对于 d=15，约为 1.5%。在标准 surface code 中，CNOT 通过 lattice surgery 实现，涉及 O(d²) 个物理门，每门错误 p_g = 0.1%，总错误 ~d² × p_g。对于 d=7，lattice surgery CNOT 错误约为 4.9%，显著高于编织方案。这说明拓扑平台的逻辑门具有天然优势。

图 3(c) 比较了实现通用量子计算的总体资源开销。对于目标门误差 ε 从 10^{-3} 到 10^{-10}：Fibonacci 直接编织方案的资源开销（所需编织操作数）随 ε 对数增长，在 ε = 10^{-10} 时仅需约 150 次编织；Majorana+魔法态蒸馏方案的资源开销（物理操作数）随 ε 更快增长，在 ε = 10^{-10} 时需要约 400 次物理操作（包括蒸馏、传输和验证）。此外，魔法态蒸馏引入了显著的**延迟**（latency）：每个 T 门需要等待蒸馏完成，而 Fibonacci 编织是即时执行的。在深度量子电路中，这一延迟差异可能更为关键。

## 6. 讨论

### 6.1 拓扑编码的核心优势

与传统 QEC 相比，拓扑编码的优势不仅是"内禀容错"——更深层次的区别在于：

1. **测量即计算**：在 Majorana 平台中，stabilizer 读出（融合测量）和逻辑门操作（编织）是同一物理过程的不同时间切片。这种"测量驱动的量子计算"（measurement-only topological quantum computation）消除了传统平台中物理门操作与纠错测量之间的硬件竞争 [11, 16]。

2. **非局域编码**：逻辑信息存储在任意子的全局拓扑构型中，而非局域的量子比特态上。这意味着即使局域的物理错误（如单个 MZM 的准粒子中毒）发生，信息也不会立即丢失——只有 anyon 对的产生-扩散-湮灭才能引入逻辑错误。

3. **渐进式容错**：在拓扑平台中，随着系统尺寸增大（更多 anyons、更大的码距），逻辑错误率趋于零的标度可能优于传统 QEC，因为拓扑保护不仅来自纠错码的冗余，还来自编织操作的几何拓扑约束。

### 6.2 当前瓶颈与解决路径

**Majorana 平台**：
- **X 环读出误差**：当前 16% 的分配误差远高于容错需求。解决路径包括：改进材料（Pb 替代 Al 已提升能隙和寿命）、优化干涉电路设计、或放弃 X 环读出而仅使用 Z 环（通过编码修改使所有 stabilizer 可通过 Z 环读出）。
- **编织验证**：Microsoft 的路线图已规划从单比特到双比特再到逻辑阵列的编织验证。本文的六篇论文（二至七）为这一过程提供了完整的理论框架和数值验证协议。

**Fibonacci 平台**：
- **实验距离**：Xu 等人 [7] 的实验是在超导处理器上模拟 Fibonacci anyons，而非在真实材料中创建。实现固态 Fibonacci anyons（如 ν = 12/5 分数量子霍尔态 [17] 或旋转双层石墨烯 [18]）仍是远期目标。
- **解码复杂性**：Fibonacci TV 码的非阿贝尔 stabilizer 需要新型解码器（如 clustering decoder 或 fusion-aware MWPM），其计算复杂度高于标准 surface code 的 blossom algorithm。

### 6.3 与前几篇论文的衔接

本文是前八篇论文的自然延伸：
- 论文一（Bi₂Se₃ 能带）→ 论文四（TI/SC MZM）→ 论文七（AM-SC 角点 MZM）提供了 Majorana 平台的材料基础；
- 论文二（Kitaev 链 MZM）→ 论文五（非绝热编织）→ 论文六（统计验证）提供了操控和验证的理论工具；
- 论文三（容错分析）→ 论文八（Fibonacci 通用编织）提供了纠错和通用计算的理论框架；
- 本文将这些工具整合为可运行的**纠错码架构**和**逻辑门路线图**。

## 7. 结论

本文系统研究了基于非阿贝尔任意子的拓扑量子纠错码与逻辑架构。主要结论如下：

1. **Tetron/Hexon 编码**：4 MZM 的 tetron 编码 1 个逻辑量子比特，6 MZM 的 hexon 编码 2 个；Majorana 表面码 4.8.8 的 stabilizer 权重为 8，每逻辑量子比特约 4d² 个 MZMs。
2. **逻辑错误率**：在 Microsoft Majorana 2 的 Z 环读出保真度 99.5% 下，d=7 的 4.8.8 码可将逻辑错误率降至 ~10^{-4}；若将 X 环读出改进至 1%，可降至 ~10^{-8}。
3. **Fibonacci 码优势**：Turaev-Viro 码在退极化噪声下阈值达 4.7%，逻辑 CNOT 通过编织实现误差仅 0.7%（d=7），远低于标准 surface code 的 lattice surgery（4.9%）。
4. **通用门资源**：Fibonacci 直接编织实现目标误差 10^{-10} 需约 150 次操作；Majorana+魔法态蒸馏需约 400 次操作，并伴随额外延迟。
5. **架构路线图**：Microsoft 的 tetron/hexon 阵列、Google 的 surface code、Quantinuum 的 color code 正在从不同路径逼近容错量子计算；拓扑平台的优势在于测量即计算、非局域编码和渐进式容错。

九篇论文构成了从材料物理到容错量子计算机的完整研究链条：拓扑绝缘体能带（一）→ Kitaev 链 MZM（二）→ 容错分析（三）→ TI/SC 实验（四）→ 非绝热编织（五）→ 统计验证（六）→ AM-SC 高阶拓扑（七）→ Fibonacci 通用计算（八）→ 拓扑纠错码与逻辑架构（九）。
