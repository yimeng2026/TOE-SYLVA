## 一、设计语言系统

### 1.1 正文穿插格式

#### 括号文学（Bracket Literature）

```
正文示例：

量子纠缠现象（别慌，不是那种纠缠）首次在实验室中被观测到。
爱因斯坦称之为"幽灵般的超距作用"（当时他的表情大概是😅）。
```

**使用规范：**

| 括号类型 | 用途 | 频率 |
|---------|------|------|
| `（）` | 口语化补充、情绪注解 | 每段1-2次 |
| `（笑` | 幽默标记 | 重要段落 |
| `（划重点` | 强调提示 | 关键概念 |
| `（小声` | 附注/冷知识 | 可选内容 |
| `（emoji` | 视觉情绪标注 | 节制使用 |

#### 颜文字与Emoji 混用规范

**科技向颜文字推荐：**

```
信息密度高: (´･ω･`)  🤔  (￣^￣)ゞ
思考/推导:   (｡･ω･｡)?  🤯  (⊙_⊙)？
突破性发现:  (o゜▽゜)o☆  🎉  ✨
严肃警告:    (｀・ω・´)  ⚠️  ❗
数据验证:    ✓( ・ω・)  ✅  🧮
翻车/意外:   (´；ω；`)  💥  😱
```

**Emoji 强度等级：**

| 等级 | 符号 | 适用场景 |
|-----|------|---------|
| 学术 | 📊 📐 🔬 | 图表、公式 |
| 科普 | 💡 🌟 ✨ | 概念解释 |
| 轻松 | 😄 🤔 🎉 | 案例、轶事 |
| 警示 | ⚠️ ❗ 🔥 | 边界条件 |

## 六、实施规范

### 6.1 设计原则

```
╔═══════════════════════════════════════════════════════════╗
║                                                           ║
║  1. 【适度原则】网聊元素是调味料，不是主菜                  ║
║     → 学术严谨性优先，趣味性服务于理解                     ║
║                                                           ║
║  2. 【层次原则】不同读者有不同体验路径                     ║
║     → 核心内容必须清晰，彩蛋可探索可跳过                   ║
║                                                           ║
║  3. 【一致原则】符号系统保持统一，建立用户预期              ║
║     → 相同图标/颜色始终代表相同含义                       ║
║                                                           ║
║  4. 【包容原则】不强制任何互动，尊重阅读偏好               ║
║     → 所有彩蛋都不影响主线阅读                            ║
║                                                           ║
╚═══════════════════════════════════════════════════════════╝
```

---

### 6.2 技术实现清单

#### 前端组件需求

```
□ TypingIndicator - 正在输入动画组件
□ MessageBubble - 消息气泡组件
□ ExpandableCard - 展开式卡片组件
□ LikeButton - 双击点赞组件
□ SlideUnlock - 滑动解锁组件
□ HiddenEasterEgg - 隐藏彩蛋组件
□ ProgressBar - 已读进度组件
□ ReferenceLink - @提及链接组件
□ MemeTermCard - 术语卡片组件
□ EmojiPicker - 表情选择器
```

#### 交互规范

```
动画时长：
- 微交互：0.2s (按钮反馈)
- 过渡动画：0.4s (页面切换)
- 加载动画：1.4s (循环)
- 彩蛋揭示：0.6s (惊喜感)

响应阈值：
- 双击间隔：< 300ms
- 长按触发：> 800ms
- 滑动手势：> 50px
- 悬停延迟：> 150ms
```
